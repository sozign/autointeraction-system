## 문제 정의

국내 블로그 플랫폼에서는 이웃 추가, 공감, 짧은 댓글과 같은
반복적이고 맥락이 약한 상호작용이 일상적으로 요구됩니다.
이러한 작업은 사용자에게 실질적인 가치나 피드백을 제공하기보다는
시간 소모적인 수작업으로 남아 있는 경우가 많습니다.

본 프로젝트는 이러한 반복 작업을 자동화하면서도,
단순한 템플릿 댓글이 아닌 게시물의 내용을 최소한으로라도 반영한
텍스트 응답을 생성할 수 있는지를 확인하기 위해 **실험적인 용도로** 시행되었습니다.

본 프로젝트의 목적은 특정 플랫폼의 성장이나 어뷰징이 아니라,
검색 → 콘텐츠 수집 → 맥락 이해 → 텍스트 생성 → 사용자 액션 실행으로 이어지는
엔드투엔드 자동화 파이프라인을 설계하고 구현하는 데 있습니다.

## 시스템 아키텍처

본 시스템은 GUI 입력을 시작점으로 하여,
인증, 검색, 콘텐츠 처리, LLM 기반 텍스트 생성,
그리고 웹 자동화 실행 단계로 구성됩니다.

전체 흐름은 다음과 같습니다.
```
사용자 입력 (GUI)
  ├─ 로그인 정보
  ├─ 검색 키워드
  ├─ 댓글 생성 방식 및 옵션
  ↓
인증 레이어
  └─ Selenium을 이용한 네이버 로그인
  ↓
검색 레이어
  └─ Search API를 통한 키워드 기반 게시물 검색
  ↓
컨트롤 레이어
  └─ 목표 이웃 수 및 반복 횟수 관리
  ↓
콘텐츠 처리 레이어
  └─ 게시물 방문 및 본문 일부 추출
  ↓
LLM 레이어
  └─ 게시물 내용을 기반으로 한 텍스트 생성
  ↓
웹 자동화 레이어
  └─ 공감, 댓글 작성, 이웃 추가 등의 사용자 액션 실행
```

## 동작 방식

1. 사용자가 GUI를 통해 로그인 정보와 검색 키워드를 입력합니다.
2. Selenium을 이용하여 계정에 로그인합니다.
3. Search API를 통해 입력된 키워드에 대한 게시물 목록을 수집합니다.
4. 수집된 게시물 목록을 순회하며 목표 이웃 수에 도달할 때까지 작업을 수행합니다.
5. 각 게시물에 대해 페이지에 접근하여 본문 일부를 추출합니다.
6. 추출된 콘텐츠를 LLM에 전달하여 댓글 생성을 시도합니다.
7. 생성된 댓글을 사용하여 게시물에 댓글을 작성하고 공감을 수행합니다.
8. 조건에 따라 이웃 추가를 진행합니다.
9. 모든 작업이 완료되면 브라우저 세션을 종료합니다.


## LLM 선택 및 프롬프트 설계

초기에는 유료 LLM API와 로컬 환경에서 실행되는 LLM을 모두 고려하였습니다.
유료 API는 한국어 성능이 우수하였으나,
지속적인 비용 발생이 부담으로 작용하였습니다.

로컬 LLM(Llama 3)은 비용 부담은 없었으나,
한국어 응답 품질이 낮고 실행 시 메모리 사용량이 커
실사용에 제약이 있었습니다.

이에 따라 비용 부담 없이 빠른 추론 속도를 제공하는
Groq 기반 LLM API를 선택하였습니다.

본 프로젝트에서는 게시물의 전체 내용을 요약하는 대신,
게시물에 등장하는 핵심 정보(예: 음식 리뷰의 경우 메뉴명)를
추출하도록 프롬프트를 설계하여,
짧고 목적에 맞는 텍스트 생성을 유도하였습니다.


```
    chat_completion = client.chat.completions.create(
      messages = [
          {
            "role": "system",
            "content": "당신은 아래 글의 내용을 읽고, 이 글에 언급된 메뉴명을 한 가지 찾는 llm입니다. 응답으로는 한국어로 된 한 단어의 메뉴 이름만을 반환합니다. 마침표, 따옴표를 포함한 문장 기호는 포함하지 않습니다. 한 단어의 한국어 메뉴명을 반환하세요."
          },
          {
            "role": "user",
            "content": f"아래 글에서 어떤 음식 메뉴를 리뷰했는지 찾고 한국어로 반환하세요. \n\n '''{half_contents}''' "
          }
        ],
        model="llama3-8b-8192",
    )
```
